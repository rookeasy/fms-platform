# FOP Brand Architecture

## Purpose

FOP uses a restrained enterprise identity system.

The brand is separated into three official assets:

- F Mark
- FOP Wordmark
- Living F

Each asset has a distinct role. No single graphic should perform every brand, lifecycle, and certification job.

## F Mark

The F Mark is the primary application icon.

Use it for:

- sidebar
- favicon
- browser icon
- mobile icon
- compact headers
- Building Library
- Mission Control
- Passport

Rules:

- simple
- timeless
- flat
- no glow
- no halo
- no text
- no embedded BUILD, ADVISE, or PROTECT labels
- recognizable at 16px, 24px, 32px, and 48px
- able to work in one colour

Colour may be applied after the mark is rendered.

## FOP Wordmark

The FOP Wordmark is the corporate identity.

Use it for:

- landing page
- login
- presentations
- website
- documentation
- proposal covers
- Passport cover

Structure:

- F Mark
- FUZION
- OPERATING PLATFORM™
- BUILD • ADVISE • PROTECT
- Buildings Under Protection™

Rules:

- do not use giant OP letters
- do not use an oversized glowing O
- do not use a halo inside the wordmark
- the company name is the wordmark
- the F is the icon

## Living F

The Living F is a lifecycle and certification mark.

It is not the logo.

It is not navigation.

It communicates:

- BUILD
- ADVISE
- PROTECT
- Protected
- Coral Halo

Use it only where lifecycle state or Passport/certification status matters:

- Building Profile
- Building Passport
- Building Library
- Mission Control lifecycle context
- Protected State
- Passport Completion

Never use the Living F as the permanent application logo.

## Living F Rules

The Living F consists of:

- top arm: ADVISE, platinum silver
- middle arm: BUILD, electric royal blue
- vertical stem: PROTECT, pastel highlighter green

The arms progressively saturate as lifecycle progress increases.

When backend Protected-State Certification is approved, and only then, the completed F receives a subtle coral halo.

The halo never appears elsewhere.

## Halo

The halo is a certification signal.

It is not decoration.

It appears only when backend Protected-State Certification is approved and the record is halo eligible.

Do not use the halo in:

- navigation
- permanent logos
- marketing wordmark
- routine loading states
- ordinary progress states

## Application Usage

Operational screens should feel like enterprise software.

The application should primarily show information, not branding.

Mission Control should feel closer to Apple Business, Autodesk Docs, or Bentley Infrastructure than a marketing page.

Use:

- white background
- deep navy typography
- electric royal blue
- platinum silver
- pastel highlighter green
- coral halo only for approved certification

Do not use:

- dark dashboards
- green canvas
- black hero sections
- giant OP logo
- glowing O
- oversized hero logos
- competing branding elements

## Marketing Usage

Marketing surfaces may use the FOP Wordmark.

Marketing surfaces include:

- landing page
- login
- website
- presentations
- proposal covers
- Passport covers

The wordmark should remain text-led and restrained.

## Passport Usage

Passport screens use:

- F Mark for application identity
- Living F for Passport lifecycle and completion state
- coral halo only when protected-state certification is approved

The certification mark belongs in the top corner or lifecycle summary, not as a page-filling logo.

## Certification Usage

Protected-state certification is backend-authoritative.

Frontend visuals must not infer the halo from visual progress alone.

The coral halo appears only when:

- protected-state status is approved
- the record is halo eligible

Suspended, revoked, unknown, or incomplete records do not show the halo.
